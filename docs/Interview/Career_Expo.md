### 14. What is MVVM architecture?

**Answer:**

"MVVM stands for Model-View-ViewModel. The View is the UI — what the user sees. The ViewModel holds the logic and state. The Model is the data layer — the database or API. The View only talks to the ViewModel, never directly to the data. This separation makes the code cleaner, easier to test, and easier to maintain. I use MVVM in all my Android apps."

---

### 15. What is an API? Have you worked with APIs?

**Answer:**

"An API is a way for two systems to communicate. One system sends a request, and the other returns a response — usually in JSON format. Yes, I've worked with APIs a lot. I've built REST APIs using Spring Boot and also consumed APIs in my Android apps — including Firebase and OpenRouter for AI features."

---

### 16. What is a REST API?

**Answer:**

"REST is a standard style of designing APIs. It uses HTTP methods — GET for reading, POST for creating, PUT or PATCH for updating, DELETE for removing data. The server is stateless, meaning each request contains all the information needed. I've built REST APIs for my SaaS products."

---

### 19. What is object-oriented programming (OOP)?

**Answer:**

"OOP is a way of writing code where we model things as objects. Each object has properties (data) and methods (behavior). The four main concepts are: Encapsulation — hiding internal details, Inheritance — one class can get features from another, Polymorphism — one interface can have multiple behaviors, and Abstraction — hiding complexity and showing only what's needed. I use OOP every day in Kotlin and Java."

---

### 20. What is the difference between ArrayList and LinkedList?

**Answer:**

"ArrayList stores elements in a fixed memory block. Accessing an element by index is fast — O(1). But inserting or deleting in the middle is slow because elements need to shift. LinkedList stores each element as a node with a pointer to the next. Insertion and deletion are fast — O(1) if you have the pointer. But accessing by index is slow — O(n). You use ArrayList when you read often, and LinkedList when you insert or delete often."

---

### 21. What is the difference between SQL and NoSQL?

**Answer:**

"SQL databases store data in tables with rows and columns — like PostgreSQL. They have a fixed structure called a schema. NoSQL databases like Firebase Firestore store data in flexible formats like documents or key-value pairs. SQL is better for complex queries and relationships. NoSQL is better for fast reads and flexible data. I've used both in my projects."

---

### 22. What is Clean Architecture?

**Answer:**

"Clean Architecture separates code into layers so that each layer has a single responsibility. In Android, I use three main layers — the data layer (database, API), the domain layer (business logic), and the presentation layer (UI and ViewModel). The key rule is that inner layers don't know about outer layers. This makes the code easier to test, maintain, and scale. I use this in Amar Batch and Amar Repair."

---

### 23. What is Jetpack Compose?

**Answer:**

"Jetpack Compose is Android's modern UI framework. Instead of writing XML layouts, you write UI in Kotlin code using functions called Composables. It's declarative — you describe what the UI should look like based on the current state, and Compose handles the updates automatically. It's faster to write, easier to read, and much better for maintaining complex UIs. All my recent Android apps use Compose."

---

### 24. What is the difference between coroutines and threads in Android?

**Answer:**

"Threads are managed by the operating system and are expensive — creating many threads uses a lot of memory. Coroutines are managed by Kotlin and are very lightweight. You can run thousands of coroutines without performance issues. Coroutines also make asynchronous code look like normal sequential code, which is much easier to read. I use Coroutines and Flow in all my Android apps."

---

### 25. What is JWT authentication?

**Answer:**

"JWT stands for JSON Web Token. When a user logs in, the server creates a token containing user information and signs it. The client stores this token and sends it with every request. The server verifies the token to confirm who the user is — without storing session data. I've used JWT in the backend of Amar Repair for secure multi-tenant user authentication."

## PART 5 — QUICK TIPS FOR TOMORROW

---

**Before the interview:**

- Greet with a handshake and a confident smile
- Say your name clearly when you introduce yourself
- Bring at least 5-10 printed copies of your CV

**During the interview:**

- Speak slowly and clearly — don't rush
- If you don't know an answer, say "I haven't worked with that specifically, but I'm confident I can learn it quickly" — don't make up answers
- Use real examples from your projects whenever possible

**Body language:**

- Sit upright
- Make eye contact
- Nod when they're speaking — show you're listening

**At the end:**

- Always thank them for their time
- Ask for a business card or LinkedIn if possible
- Say you're looking forward to hearing from them
